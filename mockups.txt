### 3. Dashboard (Developer/Company View)

*   **Page Title:** Dashboard - [Application Name]
*   **Overall Structure:** Header, Sidebar Navigation, Main Content Area.
*   **Header Components:**
    *   Logo: Text "[AppLogo]" or `<img>` placeholder.
    *   User Profile/Account `Dropdown`:
        *   Trigger: User avatar or name with `ChevronDown` icon.
        *   Items: "My Account", "Settings", `Button` (Danger or Outline): "Logout" with `LogOut` icon.
    *   Notifications: `Button` (Icon only, subtle style) with `Bell` icon. May show a `Tag` (Badge) with count for unread notifications.
*   **Sidebar Navigation Components (`w-64 bg-gray-100 p-4`):**
    *   Links styled as `Button` (full width, text-left, subtle/transparent background, hover effect):
        *   "Overview" with `LayoutDashboard` icon.
        *   "My Projects" with `Briefcase` icon.
        *   "API Keys" with `Key` icon.
        *   "Billing" with `CreditCard` icon.
        *   "Account Settings" with `Settings` icon.
        *   "Documentation" with `BookOpen` icon.
*   **Main Content Components (Overview - Default View):**
    *   Page Title: "Dashboard Overview" (`text-2xl font-semibold mb-6`).
    *   Grid of `Card` components for summary statistics:
        *   `Card` 1: "Total Projects" - Displays a number. Icon: `Briefcase`.
        *   `Card` 2: "API Usage (This Month)" - Displays a number or simple graph. Icon: `BarChart3`.
        *   `Card` 3: "Recent Activity" - List of recent events or link to activity log. Icon: `Activity`.
    *   Quick Actions:
        *   `Button` (Primary): "Create New Project" with `PlusCircle` icon.
    *   **My Projects View (Alternative view in Main Content, if "My Projects" is clicked):**
        *   Page Title: "My Projects" (`text-2xl font-semibold mb-6`).
        *   `Button` (Primary): "Add New Project" with `PlusCircle` icon, placed above the table.
        *   `Table` Component to list projects:
            *   Columns:
                *   "Project Name" (`Th`): `Td` with project name.
                *   "Status" (`Th`): `Td` with `Tag` component (e.g., `Tag` (Success) "Active", `Tag` (Warning) "Pending").
                *   "Last Updated" (`Th`): `Td` with date.
                *   "Actions" (`Th`): `Td` containing:
                    *   `Button` (Outline, small): "View" with `Eye` icon.
                    *   `Button` (Outline, small): "Edit" with `Edit3` icon.
                    *   `Button` (Danger, small): "Delete" with `Trash2` icon (possibly opens a confirmation `Modal`).
        *   `Pagination` component if the project list is long.

### 4. Project Submission Page

*   **Page Title:** Create New Project / Edit Project - [Application Name]
*   **Overall Structure:** Follows Dashboard layout (Header, Sidebar, Main Content Area).
*   **Header Components:** (Same as Dashboard)
*   **Sidebar Navigation Components:** (Same as Dashboard, with "My Projects" or relevant item potentially highlighted)
*   **Main Content Components:**
    *   A `Card` component with `p-6` or `p-8` padding contains the main form.
    *   Card Title: Dynamic - "Create New Project" or "Edit Project: [Project Name]" (`text-xl font-semibold mb-6`).
    *   Form Fields:
        *   `TextInput`: Label "Project Name". Icon: `Briefcase`.
        *   Textarea (styled like `TextInput`): Label "Project Description".
        *   `Dropdown` (Select): Label "Source Language". Icon: `Languages`. Options dynamically populated (e.g., English, Spanish, French).
        *   `Dropdown` (Multi-select or Checkbox group): Label "Target Languages". Icon: `Languages`. Options dynamically populated.
        *   `FileUploader` component:
            *   Dropzone text: "Drag & drop localization files here, or click to select."
            *   Icon: `UploadCloud`.
            *   Instruction text: "Supports JSON, YAML, XLIFF, etc."
            *   File List Preview will show uploaded files with `File` icon and `Trash2` icon to remove.
        *   Optional Advanced Settings (could be in a collapsible section):
            *   `TextInput`: Label "GitHub Repository URL (Optional)". Icon: `Github`.
            *   `TextInput`: Label "API Callback URL (Optional)". Icon: `Webhook`.
    *   Action Buttons (at the bottom of the form, typically right-aligned):
        *   `Button` (Primary): "Create Project" (if new) or "Save Changes" (if editing) with `Save` or `PlusCircle` icon.
        *   `Button` (Secondary or Outline): "Cancel".
    *   If file upload is in progress, a progress bar element (not a specific component from the list, but can be constructed with `div`s) might be shown.
    *   If there are form errors, `Alert` (Error) component might be displayed at the top of the form or individual `TextInput` components will show error states.

### 5. Translation Interface Page

*   **Page Title:** Translate: [Project Name] - [Target Language] - [Application Name]
*   **Overall Structure:** Header (specific to this interface), Main Area (Two-column layout: Source List, Translation Area), Footer/Action Bar.
*   **Header Components (Interface-specific header `p-4 border-b`):**
    *   Project Name: `text-lg font-semibold`.
    *   `Dropdown` (Select) for Target Language:
        *   Label (visually hidden or inline): "Translate to:"
        *   Icon: `Languages`.
        *   Options: List of target languages for the project.
    *   Progress Bar: Visual bar showing % completion for the selected language. (Constructed with `div`s, e.g. `w-full bg-gray-200 rounded-full h-2.5` and an inner `bg-indigo-600` div for progress).
    *   `Button` (Outline): "Back to Project Details" or "Exit Translator" with `ArrowLeft` icon.
*   **Main Content Components:**
    *   **Left Column (Source Text / Key List - `w-1/3 p-4 border-r`):**
        *   `TextInput` (Search): Placeholder "Search keys or source text..." with `Search` icon.
        *   Filter `Dropdown` (Select) or `Button` group:
            *   Options: "All", "Untranslated", "Translated", "Needs Review". Icon: `Filter`.
        *   Scrollable List of Items:
            *   Each item (`p-2 hover:bg-gray-100 cursor-pointer rounded-md`):
                *   Key Name: `text-sm font-medium text-gray-800`.
                *   Source String Snippet: `text-xs text-gray-500 truncate`.
                *   Status `Tag` (e.g., `Tag` (Primary) "Untranslated", `Tag` (Success) "Translated", `Tag` (Warning) "Needs Review").
    *   **Right Column (Translation Area - `w-2/3 p-4`):**
        *   `Card` or bordered section:
            *   Section Title: "Source Text ([Source Language])" (`text-sm font-medium text-gray-500`).
            *   Read-only display area for full source string: `p-3 bg-gray-50 rounded-md text-gray-700 min-h-[100px]`.
            *   Context/Developer Notes (if available): `text-xs text-gray-500 mt-1 p-2 bg-yellow-50 rounded-md`. Icon: `Info`.
            *   Section Title: "Translation ([Target Language])" (`text-sm font-medium text-gray-500 mt-4`).
            *   Textarea (for translation input): `block w-full p-2 border border-gray-300 rounded-md min-h-[150px] focus:ring-indigo-500 focus:border-indigo-500`.
            *   Optional: `Button` (Secondary, small): "Suggest with AI" or "Machine Translation" with `Sparkles` icon.
            *   Character limit indicator (if applicable): `text-xs text-gray-500 mt-1`.
*   **Footer/Action Bar Components (`p-4 border-t flex justify-between items-center`):**
    *   Navigation Buttons:
        *   `Button` (Outline): "Previous" with `ArrowLeft` icon.
        *   `Button` (Outline): "Next" with `ArrowRight` icon.
    *   Action Buttons:
        *   `Button` (Secondary): "Mark as Reviewed" (if applicable) with `Check` icon.
        *   `Button` (Primary): "Save Translation" with `Save` icon.
        *   `Button` (Primary, slightly different style or text): "Save and Next Untranslated" with `Save` and `ArrowRight` icons.
    *   If an error occurs during saving, an `Alert` (Error) component might be shown here or as a toast notification.
